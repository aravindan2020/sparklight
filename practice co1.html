Scenario Based Sorting Algorithm Questions
(Bloom's Level 4 – Analyze)

Bubble Sort – Space Research Mission

1. During a critical space research mission, engineers receive satellite signal-strength readings that must be arranged in ascending order to detect anomalies. However, due to limited onboard power, the system stops Bubble Sort immediately once the highest signal reaches the last position, even if remaining values are not verified. Analyze how this modified stopping condition affects correctness and determine under what conditions the data can still be considered fully sorted.
2. In a deep-space radiation monitoring system, Bubble Sort is applied but comparisons are performed only on alternate passes to reduce processor heat. Analyze how skipping comparisons affects sorting stability and identify situations where hidden disorder may still exist.
3. A space station temperature system uses Bubble Sort but swaps adjacent values only when their difference exceeds a fixed threshold. Analyze how selective swapping impacts correctness and explain whether the final output satisfies strict sorting conditions.
4. Astronaut reaction-time data is sorted using Bubble Sort, but the comparison direction is reversed after every pass. Analyze how alternating directions influence convergence and whether this modification improves or disrupts the algorithm’s behavior.

Insertion Sort – Akhanda 2

5. In Akhanda 2, warriors arrive sequentially with discipline scores, but only two shifts are allowed per insertion to maintain formation symmetry. Analyze how this restriction impacts Insertion Sort’s ability to maintain order.
6. During temple guard selection, Insertion Sort compares a new guard only with the middle of the sorted list before insertion. Analyze how this modification affects correctness and efficiency.
7. Enemy threat levels are inserted only if the incoming value is lower than the previous one. Analyze whether the list can remain sorted over time using this conditional Insertion Sort.
8. During a sacred ritual, once a priest is placed using Insertion Sort, their position cannot be changed. Analyze how this restriction affects the sorted property of the list.

Shell Sort – RRR

9. In RRR, communication priorities are sorted using Shell Sort with a fixed gap that is never reduced to 1. Analyze which elements may remain unsorted and explain why the final gap reduction is essential.
10. Secret messages are sorted using Shell Sort with randomly chosen gap values. Analyze how random gap selection affects performance and correctness.
11. British patrol timings are sorted using Shell Sort, but comparisons are restricted within regional clusters. Analyze how this affects global ordering.
12. Weapon supply data uses Shell Sort, but only half of the array is processed after each gap reduction. Analyze the impact of partial passes on disorder reduction.

Quick Sort – Baahubali

13. Warrior strengths are sorted using Quick Sort where the weakest visible warrior is always chosen as pivot. Analyze how this pivot strategy affects partition balance and performance.
14. Quick Sort recursion stops when subarray size falls below three. Analyze whether this assumption maintains correctness.
15. Battle planners apply Quick Sort but recurse on only one side of the pivot. Analyze how delayed sorting impacts total execution time.
16. Equal-strength warriors are always placed on the same side of the pivot. Analyze how this affects recursion depth and worst-case complexity.

Merge Sort – KGF

17. Gold shipment values are merged only halfway during each merge step. Analyze how premature merging affects global ordering.
18. Merge Sort uses a single reusable temporary array for all merges. Analyze how this impacts data integrity and memory usage.
19. Tunnel distances are merged with biased comparisons favoring one sublist. Analyze how this affects stability and correctness.
20. Merge Sort recursion is replaced with iteration to avoid stack overflow. Analyze whether this changes algorithmic behavior or only implementation.

Combined Sorting Algorithms – Avengers

21. Avengers’ power levels switch from Quick Sort to Bubble Sort once nearly sorted. Analyze whether this hybrid strategy improves performance.
22. Infinity Stone data is locally sorted using Merge Sort but globally combined using Insertion Sort. Analyze the scalability of this approach.
23. Threat levels are continuously inserted using Insertion Sort, but Quick Sort is applied only to the newest half of the data. Analyze whether global order is preserved.
24. Multiverse data segments use different sorting algorithms based on estimated disorder. Analyze the risks of incorrect estimation.
25. Tony Stark combines Merge Sort, Quick Sort, and Insertion Sort with switching overhead. Analyze whether the overhead outweighs theoretical gains.
