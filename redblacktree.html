Red-Black Tree

Definition

A Red-Black Tree is a self-balancing binary search tree where each node is assigned a color (Red or Black). It ensures that the tree remains approximately balanced, enabling efficient searching, insertion, and deletion operations in O(log n) time.

Why it is Colored as Red and Black

  • Colors help maintain balance of the tree
  • Red nodes cannot be adjacent → prevents skewed trees
  • Black nodes ensure equal black height on all paths
  • Helps control height without strict balancing
  • Provides faster insertion and deletion than strict balanced trees

Difference Between AVL Tree and Red-Black Tree

Feature AVL Tree Red-Black Tree
Balancing Strictly balanced Loosely balanced
Height Smaller height Slightly larger height
Rotation More rotations Fewer rotations
Insertion/Deletion Slower Faster
Searching Faster Slightly slower
Usage Read-heavy systems Write-heavy systems

When Red-Black Tree is Useful

  • When frequent insertions and deletions are required
  • Used in implementation of maps, sets, and dictionaries
  • Used in operating systems (process scheduling)
  • Used in memory management
  • Preferred in real-time applications

Rules for Forming Red-Black Tree

  • Each node is either Red or Black
  • The root node must be Black
  • No two Red nodes can be adjacent
  • Every path from root to leaf has same number of Black nodes
  • All leaf nodes (NULL nodes) are considered Black
