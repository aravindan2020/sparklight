Separate Chaining in Hashing

Definition

Separate Chaining is a collision resolution technique in hashing where each table index stores a linked list of elements. When multiple keys map to the same index, they are stored in a chain instead of probing other slots.

Algorithm (Insertion using Separate Chaining)

  1. Start
  2. Read key element k
  3. Compute index using hash function index = h(k)
  4. If slot empty → create new list and insert element
  5. If slot already contains elements → insert node at beginning/end of list
  6. Repeat for all elements
  7. Stop

C Implementation using Array of Linked Lists

#include <stdio.h>
#include <stdlib.h>

#define SIZE 10

struct node{
    int data;
    struct node* next;
};

struct node* hashTable[SIZE];

void init(){
    for(int i=0;i<SIZE;i++)
        hashTable[i] = NULL;
}

void insert(int key){
    int index = key % SIZE;

    struct node* newNode = (struct node*)malloc(sizeof(struct node));
    newNode->data = key;
    newNode->next = NULL;

    if(hashTable[index] == NULL){
        hashTable[index] = newNode;
    }else{
        struct node* temp = hashTable[index];
        while(temp->next != NULL)
            temp = temp->next;
        temp->next = newNode;
    }
}

void display(){
    for(int i=0;i<SIZE;i++){
        printf("%d :", i);
        struct node* temp = hashTable[i];
        while(temp){
            printf(" %d ->", temp->data);
            temp = temp->next;
        }
        printf(" NULL\n");
    }
}

int main(){
    int elements[10] = {23,43,13,27,33,53,63,73,83,93};

    init();

    for(int i=0;i<10;i++)
        insert(elements[i]);

    display();
    return 0;
}
