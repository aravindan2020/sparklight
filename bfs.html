Breadth First Search (BFS)

Definition

Breadth First Search (BFS) is a graph traversal algorithm that explores all the vertices of a graph level by level. It starts from a source node and visits all its immediate neighbors before moving to the next level nodes.

Why Queue is Used in BFS

BFS uses a Queue data structure because it follows the FIFO (First In First Out) principle. The first node that is discovered is processed first, ensuring level-by-level traversal of the graph.

  • Maintains order of traversal
  • Ensures shortest path in unweighted graph
  • Processes nodes level-wise

Step-by-Step Procedure to Traverse using BFS

  1. Start: Choose a starting node (source vertex).
  2. Initialize: Create an empty queue and a visited array/set.
  3. Enqueue Source: Add the starting node to the queue and mark it as visited.
  4. Repeat until Queue is Empty:
    • Dequeue a node from the front of the queue.
    • Visit (process) the dequeued node.
    • Check all adjacent (neighbor) nodes of this node.
    • If a neighbor is not visited:
      • Mark it as visited.
      • Enqueue it to the queue.
  5. Stop: When the queue becomes empty, traversal is complete.

Note: This ensures level-by-level traversal and guarantees the shortest path in unweighted graphs.

BFS vs DFS

Aspect BFS DFS
Data Structure Queue Stack / Recursion
Traversal Level-wise Depth-wise
Shortest Path Yes (Unweighted Graph) No
Memory Usage High Low
