Graph Data Structure

Definition

A Graph is a non-linear data structure consisting of vertices (nodes) and edges that connect them. It is used to represent relationships between objects.

Types of Graph

  • 1. Directed Graph: Edges have direction (A → B).
  • 2. Undirected Graph: Edges have no direction.
  • 3. Weighted Graph: Edges contain weights like cost or distance.
  • 4. Unweighted Graph: No weights on edges.
  • 5. Simple Graph: No self-loops or multiple edges.
  • 6. Multigraph: Multiple edges between same vertices.
  • 7. Connected Graph: Every node is reachable.
  • 8. Disconnected Graph: Some nodes are not reachable.
  • 9. Complete Graph: Every vertex connects to every other vertex.
  • 10. Null Graph: Nodes exist but no edges.
  • 11. Cyclic Graph: Contains at least one cycle.
  • 12. Acyclic Graph: No cycles present.
  • 13. Subgraph: Derived from another graph.
  • 14. Regular Graph: All vertices have same degree.
  • 15. Pseudograph: Allows self-loops.

Adjacency Matrix

Why?

  • Easy to implement
  • Fast edge lookup
  • Good for dense graphs

Procedure

  1. Create n × n matrix
  2. Mark 1 if edge exists
  3. Mark 0 if no edge

Transitive Closure

Why?

Used to find reachability between nodes.

Procedure (Warshall’s Algorithm)

  1. Start with adjacency matrix
  2. Check indirect paths
  3. Update matrix accordingly
  4. Repeat for all nodes

Comparison

Feature Adjacency Matrix Transitive Closure
Purpose Direct connection Reachability
Complexity O(n²) O(n³)
Usage Edge representation Path detection

Applications in Biotechnology

  • Gene interaction networks
  • Protein interaction analysis
  • Disease spread modeling
  • Drug discovery
  • DNA sequencing
